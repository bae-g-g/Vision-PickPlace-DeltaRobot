## 1. 프로젝트 개요

**팀 슈가슈가룬**의  **Delta Robot** 프로젝트입니다.<br>
이 프로젝트는 GPU가속을 이용하는 싱글컴퓨터와  MCU를 연동하여 카메라를 이용한 객체 인식 및 고속 픽앤플레이스 작업을 수행하는 델타 로봇 시스템입니다.

## 2. 팀원 및 역할

* 박시우: 시스템 설계 및 제작
* **배경근**: **델타로봇 제어 및 통신 인터페이스 구현 (역기구학, 펌웨어 및 통신 인터페이스)**
* 노영빈: 인식기술 구현, 로봇팔 연동
* 박지민: 프레임 설계 및 제작, 대시보드 페이지 제작
* 김다솜: 프레임 설계 및 제작, 데이터 수집

### 4.1 역기구학 (Inverse Kinematics)

델타 로봇의 말단 장치(End-Effector)를 원하는 좌표 `(x, y, z)`로 이동시키기 위해 각 모터의 회전 각도 `(theta1, theta2, theta3)`를 계산하는 핵심 알고리즘입니다.

- **구현 위치**: `jetson/control_delta_robot.py` 내 `DeltaRobot.inverse()` 메서드
- **알고리즘 원리**
  - 목표 위치로 부터 역으로 모터의 각도를 계산하여 델타 로봇을 제어합니다.
  - 즉, 입력된 직교 좌표계 `(x, y, z)`를 각 모터의 회전 각도로 변환합니다.
- **주요 파라미터**:
  - `R`: 베이스(Base) 반지름 (300mm)
  - `r`: 말단(End-Effector) 반지름 (60mm)
  - `L`: 상부 링크(Upper Link) 길이 (400mm)
  - `l`: 하부 링크(Lower Link) 길이 (890mm)

<details>
<summary>역기구학 계산 과정</summary>
델타로봇을

- 모터가 위치한 **베이스 → B**
- 모터에 직접 영향을 받는 **상단 링크 → L**
- 구속조건에 영향을 받는 **하단 링크 → l**
- 엔드이펙터가 위치한  **플랫폼 → P**

이렇게 크게 4가지로 구분하자

---

보편적으로 정삼각형 형태의 베이스 변의 중앙에 모터를 위치시켜서

120˚간격이 되도록한다.

마찬가지로 플랫폼 또한 정삼각형으로 형태로, 각 꼭지점을

각 모터의 수직아래 위치시켜 내접하는 형태이다.

![역기구학계산0.jpg](img/역기구학계산0.jpg)

베이스 중심점에서

- 변으로 내린 선의 길이를  $W_B$
- 꼭지점으로 내린 선의 길이를 $U_B$

플랫폼 중심점에서

- 변으로 내린 선의 길이를 $W_p$
- 꼭지점으로 내린 선의 길이를 $U_p$

라고 한다.

플랫폼이 베이스에 내접함을 가정하고

a를  베이스 한 변의 길이라고 하자

$W_B$  =  $tan30˚*a/2 = a*sqrt(3)/3$

$U_B$  =  $cos30˚*a/2 = a*sqrt(3)/6$

$W_p$  =  $W_b /2$

$U_p$  =  $U_b/2$

---

우선 기준이 되는 모터 하나에 대해서 계산하도록 한다.

→ 각 모터는 120도 간격임으로 기준모터에서 +sin30 , -sin30으로 쉽게 계산이
가능해진다.

![역기구학계산1.jpg](img/역기구학계산1.jpg)

---

현재 목표는 P가 특정 위치에 존재하기 위한 각 모터의 회전 각도를 구하는 것이다.

특정 위치를 구하기 위해  3차원 벡터형태로 각 델타로봇의 성분들을  나타내면

$B^B = <0,-W_B,0>$    베이스중심 →  모터1

$L =< 0, L*sinθ, -L*cosθ>$   모터 →  상단링크 ( L = 링크의 길이 = 상수)

- ※ x축 기준으로 회전하는 모터의 움직임(=θ)에 따라  y축,z축으로의 이동방향이 정해진다.

$l =?$     상단링크 → 하단링크

$P_P^B = <x,y,z>$   베이스중심  → 플랫폼중심

$P^P=<0,-U_p,0>$  플랫폼중심 → 하단링크

①과 같이 벡터 합의 표현으로 나타내면 좌변과 우변이 같다.

이 성질을 이용하여  $l$에 대한 식으로 다시 표현이 가능하다.

③ 벡터를 거리에 대한 식으로 나타내었을때  $l$의 길이는 정해진 상수임으로

제곱으로 풀어낸다면

$x^2 + y^2 + z^2 α^2 + L^2 -l^2 +2yα -(2αL + 2yL)sinθ + (2zL)cosθ$

(  $α$  =  $-U_p + W_B$    )

$x^2 + y^2 + z^2 α^2 + L^2 -l^2 +2yα$

= $C$

$-(2αL + 2yL)$

= $A$

$(2zL)$

= $B$

⇒      $C + Asinθ + Bcosθ$

여기서 θ를 제외한 모든 값은 상수임

![역기구학계산2.jpg](img/역기구학계산2.jpg)

θ를 구하기 위해 tan 반각 공식을 이용하여 치환한다.

$tan(θ/2) = t$

$sinθ = 2t/(1+t^2)$

$cosθ = (1-t^2)/(1+t^2)$

④t에 대한 식으로 변형 후 근의 공식으로 t의 해를 얻었다면

arctan을 이용해 θ값을구하는것이 가능해진다.

- θ는 각 링크가 베이스에서 플랫폼으로 수직으로 내려진 각도를 0도라고  가정하여
  약 -90˚ ~ 0˚정도의 값을 갖는다. 근의 공식을 통해 구해진 두 개의 값 중 적절한 조건으로
  올바른 θ값을 구한다.

나머지 두 개의 모터 각도 또한 위와 같은 방식으로 구해진다.

### 4.2 펌웨어 (Firmware Implementation)

Arduino 기반의 펌웨어는 실시간 모터 제어와 센싱을 담당합니다.

- **개발 환경**: Arduino IDE / PlatformIO
- **핵심 라이브러리**:

  - `AccelStepper`: 스텝 모터의 부드러운 가감속(Acceleration/Deceleration) 제어.
  - `TaskScheduler`: 멀티태스킹으로 구현하여 명령어 수신, 처리, 모터 구동을 병렬적으로 수행.
- **주요 기능**:

  1. **속도 동기화 (Speed Synchronization)**:
     - 3개의 축이 목표 지점에 동시에 도달하도록 각 축의 최대 속도와 가속도를 동적으로 조절합니다.
     - `SpeedSynchronization()` 함수에서 가장 긴 이동 거리를 기준으로 비율을 계산하여 적용합니다.
  2. **명령어 큐 (Command Queue)**:
     - Jetson으로부터 수신된 명령어를 큐(Queue)에 저장하여, 현재 동작 중에도 다음 동작을 예약할 수 있습니다.
     - `myQueue` 클래스를 사용하여 FIFO 방식으로 명령을 관리합니다.
  3. **인터럽트 처리 (Interrupt Handling)**:
     - 외부 센서(진공 센서 등) 입력을 인터럽트로 처리하여 즉각적인 반응(비상 정지, 모드 전환 등)을 보장합니다.

### 4.3 통신 인터페이스 (Communication Interface)

Jetson Nano(싱글보드 컴퓨터)와 teensy4.1(mcu) 간의 빠르고 신뢰성 있는 데이터 전송을 위해 **Serial Communication (USB)**을 사용합니다.

- **구현 위치**:

  - **Jetson (Sender)**: `jetson/control_delta_robot.py`
  - **teensy4.1 (Receiver)**: `fiarmware/main.cpp`
- **통신 설정**: `Baudrate: 115200`
- **명령어 프로토콜 (Protocol)**:
  상위 제어기에서 아래와 같은 문자열 명령어를 전송하여 로봇을 제어합니다.


  | 명령어    | 형식               | 설명                                         | 예시                       |
  | :-------- | :----------------- | :------------------------------------------- | :------------------------- |
  | **Move**  | `M <p1> <p2> <p3>` | 3개의 스텝모터를 지정된 위치(Step)로 이동    | `M 1000 1000 1000`         |
  | **Speed** | `S <s1> <s2> <s3>` | 각 모터의 최대 속도 설정                     | `S 2000 2000 2000`         |
  | **Grip**  | `G <1,0>`          | 그리퍼(진공 펌프/서보) 제어 (1: ON, 0: OFF)  | `G 1` (잡기), `G 0` (놓기) |
  | **Reset** | `R`                | 호밍(Homing) 및 초기 위치 보정, 캘리브레이션 | `R`                        |
  | **Down**  | `D`                | 하강 모드 진입 (인터럽트 기반 제어 준비)     | `D`                        |
- **동작 흐름**:

  1. Jetson에서 객체 인식 후 목표 좌표 계산 및 역기구학 풀이.
  2. 계산된 스텝 값을 `M` 명령어로 Arduino에 전송.
  3. Arduino는 `AccelStepper` 라이브러리를 통해 부드러운 가감속 제어로 모터 구동.
  4. 그리퍼 동작 시 `G` 명령어로 진공 펌프 제어.

### 앤드이펙터 좌표 오차 및 보간

**1. 문제 상황**
엔드이펙터의 실제 위치가 프로그램에서 계산한 목표 좌표값과 일치하지 않는 현상이 발생했습니다.

- **예상 원인 1**: 카메라 렌즈 왜곡에 의한 좌표 오차
- **예상 원인 2**: 수작업으로 제작한 3D 프린팅 감속기의 물리적 오
- **예상 원인 3**: 역기구학 수식의 계산 오류

**2. 원인 분석 및 파악**

- **카메라 왜곡 검증**:
  - 만약 카메라 렌즈의 왜곡(Radial Distortion)이 원인이라면, 정형적인 패턴으로 오차가 발생해야 했습니다.
  - 하지만 실제 측정된 오차는 위치와 무관하게 불규칙적으로 발생했으므로, 렌즈 왜곡이 주원인이 아님을 확인했습니다.
- **실제 원인 **:
  - 장시간 반복 테스트 결과, 감속기의 장력 풀림과 급격한 기동 시 발생하는 진동이 오차의 핵심 원인임을 확인했습니다.

**3. 해결**

- ~~신경망을 이용한 보정~~

  - 델타 로봇이 이동 할 수 있는 모든 좌표를 이동후 카메라에 찍힌 좌표와 프로그램에서 계산한 좌표를 저장하는 스크립트를 만들고 신경망을 학습시킴
    - z좌표의 보정은 실제로 10mm정도 오차를 기록하며 보간되는듯 하였으나 x,y좌표로 이동의 오차는 30mm이상 기록하며 보간되지 않는 현상이 발생함
    - 원인은 감속기의 구조로, x-y축으로 이동시 각 감속기가 받는 장력이 각기 달라지기 때문에 발생하는 불규칙적 오차
- **실시간 아루코마커를 통한 보정**

  - 속도라는 trade-off를 감수하고 실시간으로 아루코마커를 통해 좌표를 보정하는 방식.
  - 물체가 이동해야하는 위치를 목표지점으로 실시간으로 엔드이펙터의 마커를 추적하며 목표지점과 엔드이펙터의 마커의 오차만큼의 거리를 보정하는 방식을 채택함
  - 그리핑 과정에서 1초 정도의 시간이 추가적으로 소요되지만, 100%에 가까운 성공률을 보임
